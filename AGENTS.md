# local-typeless — agent entry point

Native macOS menu-bar dictation app. Tap a global hotkey to record; audio is transcribed locally (EN + ZH) by WhisperKit, rewritten by an on-device MLX LLM, and pasted into the focused app. Fully offline after initial model download.

## Stack

- Swift 5.9, SwiftUI + AppKit, macOS 14+ deployment target
- `SWIFT_STRICT_CONCURRENCY: complete` — most pipeline state is `@MainActor`-isolated
- [WhisperKit](https://github.com/argmaxinc/WhisperKit) for ASR (`whisper-large-v3-turbo` CoreML)
- [MLX Swift Examples](https://github.com/ml-explore/mlx-swift-examples) for LLM polish (`Qwen2.5-3B-Instruct-4bit`)
- [GRDB](https://github.com/groue/GRDB.swift) for SQLite history persistence
- Carbon `RegisterEventHotKey` + `CGEventTap` for hotkeys
- AVAudioEngine for capture; `NSPasteboard` + `CGEvent` for paste injection
- Xcode project is generated by [XcodeGen](https://github.com/yonaskolb/XcodeGen) from `project.yml` — **never edit `LocalTypeless.xcodeproj/` by hand**

## Dev commands

```sh
make bootstrap-signing # ensure the stable local Keychain signing identity exists
make generate   # regenerate LocalTypeless.xcodeproj from project.yml
make build      # xcodebuild build -scheme LocalTypeless
make test       # xcodebuild test (LOCAL_TYPELESS_SKIP_MODEL_TESTS=1)
make run        # launch the built .app
make install    # install, sign, and open /Applications/LocalTypeless.app
make clean      # wipe build/, DerivedData, generated xcodeproj
```

After editing `project.yml`, run `make generate` before building. The checked-in `LocalTypeless.xcodeproj/project.pbxproj` is also regenerated by `make build` / `make test`; avoid committing the churn unless `project.yml` actually changed.

## Local sync (Syncthing)

The maintainer mirrors this repo across LAN machines via Syncthing, with the repo root as the folder root. A few rules follow from that:

- **`.stignore` is per-machine config** — git-ignored (see `.gitignore`'s `Syncthing` block) and not synced by Syncthing itself. Different peers may have different storage realities (one cares about disk space and skips `Models-Cache/`, another doesn't), so each LAN peer creates its own `.stignore` from the template below.
- **`.git/` is excluded on purpose.** Concurrent multi-machine writes corrupt the git index. Use `git push` / `git pull` against the GitHub remote for cross-machine git state — Syncthing handles only the working tree.
- **All generated artifacts are excluded.** That includes `*.xcodeproj/` and `*.xcworkspace/` (regenerated by `make generate` / `make generate-portable`), `build/`, `DerivedData/`, `.build/`, `Packages/`, `Package.resolved`, `.swiftpm/`, plus the user-state files Xcode drops into these (`xcuserdata`, `*.xcuserstate`, etc.). Regenerating locally is cheap and avoids syncing hundreds of MB of churn.
- **Agent state under `.claude/` is excluded.** Worktrees, scratch files, agent memory — all per-machine. Real work moves between peers through the git remote, not through filesystem sync. If you want cross-machine continuity for an in-progress agent task, commit + push + check out on the other peer.
- **Use `**/` wildcards, not root-anchored `/`.** Build artifacts can appear inside nested workspaces (e.g. an agent worktree that wasn't ignored yet, or a Swift package sub-folder). `**/build` catches them all; `/build` only catches the top-level one. This is the most common ignore-pattern bug.
- **`(?d)` marks "delete on appearance".** If a matching file ever shows up locally (someone copied a `build/` folder in, or the previous `.stignore` was too permissive), Syncthing removes it. Aggressive but appropriate for purely-generated artifacts.
- **New ignorable artifacts go in `.gitignore` first**, then mirror into the template below. Syncthing's `.stignore` doesn't `#include .gitignore` (gitignore's `#` comment syntax collides with Syncthing's pattern grammar), so the two lists are kept in sync by hand.

## Coding conventions

- **Concurrency.** Pipeline state lives on `@MainActor`. Services crossing actor boundaries (ASR, polish, audio tap closures) must be `Sendable`. The `AudioBuffer` class is thread-safe via `NSLock` and is captured directly into non-main tap closures.
- **Protocols for swappable backends.** `ASRService`, `PolishService`, `TextInjector`, `HistoryStore`, and the `*ModelManaging` protocols exist specifically so tests inject stubs. Prefer extending a protocol over reaching into a concrete type.
- **Errors.** Pipeline errors flow as `DictationPipeline.Event.failed(String)` — UI maps them to `StateMachine.fail(message:)`. Injection / polish / audio-save failures degrade gracefully (log + continue) rather than aborting the whole flow.
- **No network calls in the dictation path.** The only networked work is one-time model download from HuggingFace via WhisperKit / MLX download APIs, gated behind an explicit user action.
- **Logging.** Use `Log.<category>` from `Support/Logger.swift`. Never print secrets; mark dynamic fields `privacy: .public` only when the content is safe.
- **Localization.** Every user-visible string goes through `String(localized:)` and lands in `Resources/Localizable.xcstrings` with both `en` and `zh-Hans`. `LocalizationCoverageTests` asserts full coverage.

## Things that trip up agents here

- **`AudioBuffer` is ambiguous** in code that transitively imports CoreAudio — CoreAudio exposes a C struct of the same name. In that context, qualify as `LocalTypeless.AudioBuffer`. The test target hits this via bridging-header imports.
- **Hardware paste is layout-dependent.** `TextInjector.postCommandV` uses the *physical* V key code (`kVK_ANSI_V`), not the character `"V"`, to survive layouts that remap QWERTY under ⌘. Don't "simplify" that back to a character.
- **Push-to-talk only works on modifier-only `.press` bindings.** For any other binding shape `HotkeyMode.effective(for:)` silently falls back to `.toggle` — a Carbon key-down event has no meaningful "release". See `Core/HotkeyMode.swift`.
- **Model status is not the same as product policy.** `ModelStatusStore` only says where a model is in its lifecycle (`downloaded`, `loading`, `resident`, etc.). User-facing behavior also depends on `AppSettings.polishMode` and `MemoryAdvisor`. When changing model lifecycle, polish behavior, memory gating, or wording, update all connected surfaces together: `AppDelegate` gating/skip/unload logic, `DictationPipeline.Input.polishEnabled`, `StateMachine` transitions, `MenuBarController` status labels, `SettingsAdvancedTab`, `ModelDownloadView`, `docs/architecture/models.md`, and every `Localizable.xcstrings` key in both `en` and `zh-Hans`.
- **Downloaded means on disk; ready means in memory.** Never label `.downloaded` as loaded/ready in UI. Use language like "ready on demand" for downloaded-but-not-resident models, and "ready (in memory)" only for `.resident`.
- **Polish is optional, ASR is mandatory.** In `automatic` or `off`, missing or memory-skipped polish must not block recording. If polish is disabled or memory policy skips it, unload a resident polish model so the menu, RAM usage, and behavior stay aligned.
- **Normalize final text, not just raw fallback text.** Whisper may insert spaces between Chinese words, and polish output can preserve or introduce them. Run `TranscriptTextNormalizer.finalOutput(_:transcript:)` after successful polish as well as on skipped/failed polish paths before injection and history persistence.
- **The listening HUD is part of the recording contract.** `RecordingHUDController` follows `StateMachine` states and uses `AudioLevelMeter` fed by `Recorder` to show real live waveform bars and elapsed time while recording. Keep it pointer-transparent and above normal apps so users can tell the hotkey started listening without the HUD stealing focus. If you change recorder initialization, verify the meter is still wired.
- **Accessibility fallback is not paste success.** Auto paste requires macOS Accessibility trust. Local debug builds are ad-hoc signed unless a signing identity exists, and replacing `/Applications/LocalTypeless.app` can make TCC distrust the app again. If `TextInjector` reports `.accessibilityDenied`, leave text on the pasteboard and provide a direct "Open Accessibility Settings" action; do not treat this as a focused-window failure.
- **Preserve the whole pasteboard, not only strings.** When `preserveClipboard` is enabled, `TextInjector` snapshots every pasteboard item/type/data payload before staging dictation text and restores that full snapshot after a successful paste. Do not regress this to storing only the first `.string`; users may have images, RTF, files, or multiple pasteboard items.
- **Stable signing matters for local installs.** TCC grants are tied to the app's code requirement. Ad-hoc rebuilds change the cdhash, so Accessibility/Input Monitoring/Microphone grants can appear enabled in System Settings while TCC still rejects the current binary. Run `make bootstrap-signing` once to ensure the `Glossa Local Dev Code Signing` identity exists in the user's login Keychain, then use `make install`. Re-grant permissions for `/Applications/LocalTypeless.app` once after creating or changing that identity.
